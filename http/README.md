## URI와 웹 브라우저 요청 흐름 
* Uniform: 리소스 식별하는 통일된 방식
* Resource: 자원, URL로 식별할 수 있는 모든 것
* Identifier: 다른 항목과 구분하는데 필요한 정보
* URL과 URI는 별차 없다.

## URL scheme
* 프로토콜 : http, https, ftp 등
* 호스트명 : 도메인명
* 포트 : PORT (8080)
* path: 리소스 경로, 계층적 구조
* Query: key=value 형태, 웹서버에 제공하는 파라미터, 문자형태

## 클라이언트 서버 구조
* Request Response 구조
* 클라이언트는 서버에 요청을 보내고, 응답을 대기
* 서버가 요청에 대한 결과를 만들어서 응답
